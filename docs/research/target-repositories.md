# Target Repository Baseline

## First-round POC

### RuoYi-Vue

- Repository: `yangzongzhuan/RuoYi-Vue`
- Stack: Java / Spring Boot / Maven / Vue
- Role: Java enterprise admin-system baseline
- Why: clear modules, common B-side architecture, lightweight enough for first POC

### eShopOnWeb

- Repository: `dotnet-architecture/eShopOnWeb`
- Stack: .NET / ASP.NET Core / EF Core
- Role: non-Java cross-stack baseline
- Why: clean architecture, clear project layout, tests available, low runtime cost

## Second-stage candidate

### Apache Fineract

- Repository: `apache/fineract`
- Stack: Java / Spring Boot / Gradle / PostgreSQL
- Role: complex business-system validation candidate
- Why: real financial system, large module graph, strong enterprise/business representativeness
- First-round status: static baseline only; no full build target
